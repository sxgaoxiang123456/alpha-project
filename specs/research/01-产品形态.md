### 2.2 首页信息架构详解

#### 同花顺APP首页（低密度、大字显示）

```
┌─────────────────────────┐
│  搜索栏 + 问财AI入口     │  ← 接入DeepSeek-R1，自然语言搜索
├─────────────────────────┤
│    大盘指数核心行情区     │  ← 大字、少信息、高对比度
├─────────────────────────┤
│   自选股价格走势预览区    │  ← 分时预览，右滑切换列表/文字
├─────────────────────────┤
│    功能矩阵（可自定义）    │  ← 长按拖动、隐藏
├─────────────────────────┤
│     资讯/公告信息流       │  ← 相对弱化
└─────────────────────────┘
```

**设计原则**：减法设计，降低认知负荷，聚焦核心交易功能。

#### 东方财富APP首页（高密度、信息量大）

```
┌─────────────────────────┐
│  搜索栏 + 个性化快捷入口  │  ← 支持自定义配置
├─────────────────────────┤
│   资讯瀑布流（4-5篇）    │  ← 实时快讯、权威分析优先
├─────────────────────────┤
│    热门股吧/社区动态      │  ← 社区互动核心入口
├─────────────────────────┤
│   行情指数+自选股概览    │  ← 信息密度高，数字较小
├─────────────────────────┤
│  基金/理财/交易快捷入口   │  ← 向证券、基金交易导流
└─────────────────────────┘
```

**设计原则**：加法设计，一站式满足看、聊、买需求；高度可定制化（板块顺序、显示/隐藏均可调）。

#### 雪球APP首页（社区+内容驱动）

```
┌─────────────────────────┐
│  顶部"我的"入口 + 搜索栏  │  ← "我的"从底部导航移出
├─────────────────────────┤
│   推荐页 + 8个快捷入口   │  ← 选股、选基触手可及
├─────────────────────────┤
│    视频页（短视频/直播）  │  ← 热门排行、每周必看
├─────────────────────────┤
│   选股攻略 + 基金必看     │  ← 50只热股"前世今生"
├─────────────────────────┤
│    社区UGC内容流         │  ← 用户讨论、跟投动态
└─────────────────────────┘
```

> **来源**：雪球13.0新版改版说明 [[6]](https://xueqiu.com/7656754363/198917094)

### 2.3 自选股/行情/推送的组织方式

| 产品 | 自选股组织 | 行情组织 | 推送/通知组织 |
|:---|:---|:---|:---|
| **同花顺** | 列表+分组+分时预览；右滑切换文字/图表 | 首页大盘+行情Tab全市场；Level-2十档免费 | APP内消息中心；智能盯盘提醒；桌面小组件 |
| **东方财富** | 自选Tab独立；资讯/公告/研报/财务一级直达 | 行情Tab含指数/板块/个股；数据覆盖全面 | 股吧动态推送；自选异动提醒；基金净值通知 |

| **腾讯自选股** | 自选Tab为核心；未登录态可用；实时推送报价 | 行情Tab为市场总览；简洁展示 | 股价提醒（2步设置）；锁屏推送 |

## 三、交互模式：自选股管理、智能选股、异动推送

### 3.1 自选股管理交互套路

| 交互维度 | 常见设计模式 | 代表产品 |
|:---|:---|:---|
| **添加自选** | 搜索代码/名称 → 点击"+自选"；扫码添加；批量导入 | 全产品通用 |
| **分组管理** | 长按拖动排序；新建/重命名/删除分组；股票拖拽归类 | 雪球、中广云智投 |
| **列表展示** | 价格+涨跌幅+涨跌额三列；右滑切换分时/K线/文字 | 同花顺、东方财富 |
| **数据维度** | 行情/资金/公告/研报一体化；点击展开详情 | 新浪财经、东方财富 |
| **跨端同步** | QQ/微信登录自动同步；云端合并本地列表 | 腾讯自选股、雪球 |
| **快捷操作** | 左滑删除/置顶；长按多选；批量操作 | 主流APP通用 |

> **来源**：雪球自选股功能升级 [[8]](https://xueqiu.com/3771154167/20591617)；腾讯自选股设计分析 [[9]](https://www.woshipm.com/evaluating/341851.html)

### 3.2 智能选股交互套路

| 交互模式 | 形态描述 | 代表产品 |
|:---|:---|:---|
| **自然语言选股** | 搜索框输入白话条件，如"MACD金叉，均线多头排列" | 同花顺问财、AI涨乐 |
| **条件筛选器** | 多维度下拉菜单：技术面/基本面/资金面/市值/行业 | 同花顺、东方财富、广发易淘金 |
| **形态识别选股** | K线图上框选一段走势 → 自动匹配相似形态股票 | 同花顺（快捷键`79`） |
| **策略回测** | 设置选股条件 → 历史数据回测 → 生成胜率报告 | 同花顺问财、广发金股雷达 |
| **AI推荐选股** | AI主动推荐热点/涨停猎手/动态估值工具 | AI涨乐、广发易淘金 |
| **语音选股** | "帮我找形态类似XX的股票" | 广发易淘金i助理 |

**广发易淘金"股市天眼"工具矩阵**：
> 股哥盯盘、分析师评股、择时决策、解禁地雷、专题选股、多空情绪、投资日历、**形态选股**、相似K线、商誉报告、涨停分析、筹码成本、大盘异动、指数估值、南北资金流向等17+工具。

> **来源**：同花顺形态选股教程 [[10]](http://www.360doc.com/content/24/0130/17/27524644_1112714788.shtml)；广发易淘金智能选股 [[11]](https://jingyan.baidu.com/article/636f38bb7e85ca97b9461075.html)

### 3.3 异动推送交互套路

| 交互环节 | 常见设计模式 | 代表产品 |
|:---|:---|:---|
| **预警设置** | 价格/涨跌幅/技术指标/量能/资讯多维度条件 | 同花顺、东方财富、e海通财 |
| **自然语言设预警** | "某股票跌破1400元时提醒我" | 广发易淘金i盯盘、AI涨乐 |
| **分级触达** | Level1观察（仅记录）→ Level2关注（标准Push）→ Level3预警（强提醒+声音/震动） | OpenClaw方案、主流APP |
| **触发后动作** | 推送 → 点击展开详情 → 查看个股 → 加入自选/快速下单 | 同花顺、华泰AI涨乐 |
| **盯盘时段** | 仅交易时段（9:30-11:30, 13:00-15:00）或7×24小时 | AI涨乐（7×24）、传统工具（交易时段） |

> **来源**：券商APP AI重构分析 [[12]](http://mp.weixin.qq.com/s?__biz=MzkzNjYzNzE4NA==&mid=2247509701&idx=1&sn=6f11a9e9cb00b51efbce17feb3937f14)；OpenClaw盯盘工具 [[13]](https://developer.aliyun.com/article/1720828)

## 五、推送形态：早盘简报/异动预警怎么呈现？

### 5.1 业内常见推送渠道矩阵

| 渠道 | 适用场景 | 形态特征 | 代表案例 |
|:---|:---|:---|:---|
| **APP Push** | 个人实时交易决策 | 标题+摘要+展开详情；支持分级触达（观察/关注/预警） | 同花顺、东方财富、华泰AI涨乐 |
| **飞书卡片消息** | 团队共享/投研协作/B端风控 | 结构化卡片：标题+指标区+图表+操作按钮 | OpenClaw方案、金融机构内部 |
| **企业微信图文** | 投顾-客户触达/内部通讯 | 图文消息/Markdown/图文卡片 | 券商投顾服务 |
| **公众号图文** | 投资者教育/品牌内容运营 | 长图文排版、早报、复盘 | 各大券商/基金公司公众号 |
| **短信** | 强提醒/合规通知 | 纯文字，字数限制 | 价格预警、新股中签 |
| **桌面小组件** | 碎片化看盘 | 实时价格/涨跌幅小窗 | 腾讯自选股、同花顺 |
| **语音播报** | 开车/运动场景 | TTS语音播报早盘要点 | 华泰AI涨乐"早点听" |

### 5.2 APP Push 推送形态详解

**内容结构（以异动预警为例）**：

```
【标题】🔥 异动预警：XX股份（600XXX）5分钟涨速超5%
【副标题】成交量放大3倍，主力资金净流入XXXX万
【展开内容】
- 当前价：XX.XX元 (+X.XX%)
- 异动类型：急速拉升 / 放量突破 / 盘口异动
- 关联板块：XXX概念
【跳转动作】打开个股详情页 / 交易页面
```

**分级触达设计**：

| 层级 | 触发条件 | Push形态 |
|:---|:---|:---|
| Level 1 观察级 | 异动评分较低 | 仅记录，不推送 |
| Level 2 关注级 | 达到预警阈值 | 标准Push通知（标题+摘要） |
| Level 3 预警级 | 高置信度异动 | 强提醒Push + 声音/震动 + 展开详情 |

> **来源**：OpenClaw盯盘实战 [[13]](https://developer.aliyun.com/article/1720828)

### 5.3 飞书卡片消息形态

**典型早盘简报卡片结构**：

```
📊 A股早盘简报 | 202X年XX月XX日
─────────────────────────
【大盘概览】
上证指数 XXXX.XX  +X.XX%
深证成指 XXXX.XX  +X.XX%
创业板指 XXXX.XX  +X.XX%
─────────────────────────
【异动预警】
🔴 XX股份(600XXX) 涨速+5.2%  成交量突增
🟡 YY科技(002XXX) 放量突破   主力资金流入
─────────────────────────
【板块热点】
1. 半导体  +X.X%  （政策利好驱动）
2. 新能源  +X.X%  （业绩超预期）
─────────────────────────
[查看详情] [加入自选] [设置提醒]
```

**飞书卡片技术特征**：
- 支持富文本、颜色标识（红涨绿跌）、按钮跳转
- 可通过飞书机器人/Bot自动推送
- 支持@用户、群组定向推送
- JSON格式配置，可嵌入多维表格数据

> **来源**：飞书消息卡片开发文档 [[26]](https://blog.csdn.net/luohualiushui1/article/details/155771066)

### 5.4 公众号图文形态

**早盘简报公众号图文典型结构**：

```
标题：【早盘必读】XXXX年XX月XX日 | 三大指数高开，半导体板块领涨

封面：大盘涨跌数据可视化图表

正文：
1. 隔夜外盘回顾（美股/欧股/亚太）
2. 早盘数据概览（涨跌停家数、成交额、北向资金）
3. 板块轮动分析（领涨/领跌板块+原因）
4. 个股异动精选（3-5只重点股票+逻辑）

6. 风险提示

底部：免责声明 + 关注引导 + 历史回顾链接
```

### 5.5 推送形态设计建议（针对A股自动盯盘AI助手）

| 场景 | 推荐渠道 | 形态 | 原因 |

## 八、调研结论与产品启示

### 8.1 对A股自动盯盘AI助手的产品启示

1. **首页设计**：建议采用"低密度+核心信息前置"模式（同花顺路线），自选股和大盘指数打开即见，降低认知负荷。

2. **AI入口**：推荐"搜索框即AI"（同花顺问财模式）+ "预判式服务卡片"（妙想模式）双轨并行，既支持主动查询，也支持被动推送。

3. **自选股管理**：必备分组管理、批量操作、跨端同步；可探索"AI智能分组"（根据行业/概念/走势自动归类）。

4. **智能选股**：自然语言选股已成标配（问财模式），形态识别是差异化亮点（同花顺形态选股），策略回测增强可信度。

5. **异动推送**：必须分级触达（避免信息过载），支持多渠道（APP Push+飞书卡片+语音播报），自然语言设预警是体验升级点。

6. **早盘简报**：飞书卡片适合B端/团队协作场景；公众号图文适合C端/品牌运营；APP内嵌适合个人交易决策。

### 8.2 行业趋势判断

- **2025-2026年核心趋势**：从"功能叠加"进入"智能原生"时代，LUI+GUI双驱动成为标配。
- **技术投入**：证券行业信息技术投入预计突破500亿元。
- **垂直模型替代通用模型**：金融专用大模型（HithinkGPT、妙想、天玑智融）正在替代通用LLM。
- **全场景闭环**：选股→盯盘→交易→风控→陪伴的完整智能服务链是终极目标。
