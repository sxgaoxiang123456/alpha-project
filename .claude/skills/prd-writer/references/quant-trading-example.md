## 2. 项目背景与目标

### 2.1 背景
我每天上班 996，根本没时间盯盘。但身边同事最近都在聊股票，A 股也时不时来一波，我心里痒——想做点投资又不想 all in。问题是每天能花在股市的时间只有早 10 分钟、午 10 分钟、晚半小时。靠这点时间手动看盘根本不够。

调研发现：
- 同花顺 / 东方财富功能太重，信息淹没
- 雪球本质还是手动浏览
- 聚宽 / 米筐 / QuantConnect 门槛太高
- ChatGPT 不知道实时行情

**空白点**：没有一个"AI 个性化盯我的自选股 + 主动推送 + 我能自己改造"的工具。

### 2.2 目标（Goals）
- **业务目标**：让我每天少花 30 分钟在股市信息收集上
- **产品目标**：用户从"知道某只股票异动"的时长，从平均 30 分钟缩短到 1 分钟内
- **学习目标**：验证"AI 自然语言转选股查询"在中文场景下的可用性

### 2.3 非目标（Non-Goals）
- ❌ **不做自动交易** —— 涉及真钱、合规复杂
- ❌ **不做策略研究** —— 我不是量化研究员
- ❌ **不做社区/分享** —— 这是个人工具
- ❌ **不做美股/港股/期货/加密** —— 只做 A 股，专精
- ❌ **不做移动端 APP** —— 网页版 + 飞书推送就够
- ❌ **不做付费墙** —— 给自己用，不商业化

### 2.4 战略对齐
作为"真实企业级 0→1 项目"的实战案例，验证 Vibe Coding 方法论的端到端可行性。

### 3.1 主要画像 P1：996 职场技术散户
- **背景**：30-40 岁，IT 行业，会编程，懂基本金融概念
- **目标**：用最少时间获取最关键的股市信息，辅助投资决策
- **痛点**：
  - 上班没时间盯盘
  - APP 信息过载
  - 想要个性化推荐而非千篇一律
  - 重要异动总是错过
- **使用频次**：每天 3-5 次（开盘前 / 午盘 / 收盘 / 晚上复盘）
- **设备**：MacBook（主）+ iPhone（推送接收）+ 飞书
- **数字代表**：愿意花 ¥100/月养工具，但前提是能省 30 分钟/天

## 4. 用户故事

### US-01 早盘信息汇总
作为 996 职场散户
我希望 早上 8:00 在飞书收到一份个性化早盘简报
以便 通勤路上 5 分钟看完，到公司就能开盘

**触发情境**：周一到周五早上 8:00（避免周末）
**期望结果**：用户看完简报后，对今日市场重点和自选股状态有清晰认识

### US-02 实时异动预警
作为 工作日忙得不可开交的散户
我希望 我的自选股有重大异动时，飞书立即推送
以便 在 1 分钟内得知，决定是否处理

**触发情境**：交易时段（9:30-15:00），自选股涨跌 > 5% 或有重大公告
**期望结果**：30 秒内收到推送，包含异动原因 + 建议关注点

### US-03 自然语言选股
作为 不会写复杂量化代码的散户
我希望 用大白话搜索股票（"最近 3 天放量上涨的新能源股"）
以便 不用学复杂的筛选器语法，几秒就能找到目标

**触发情境**：周末复盘 / 看到新闻产生灵感 / 寻找投资机会时
**期望结果**：3 秒内出结果，准确率 ≥ 80%

### US-04 自选股监控
作为 持有 10-30 只股票的散户
我希望 一屏看完所有自选股的实时状态
以便 不用打开 N 个 APP 切来切去

**触发情境**：每天早上和午休时打开
**期望结果**：1 屏装下，关键指标（价格、涨跌、量比）一目了然

### US-05 周度复盘
作为 有学习意识的散户
我希望 每周日自动收到一份"我的一周投资复盘"
以便 总结教训，优化下周决策

**触发情境**：每周日晚上 20:00
**期望结果**：包含本周自选股表现 + AI 摘要 + 关键事件

## 6. 详细功能描述

### F1 自选股监控 Dashboard

**触发**：用户在浏览器打开网页

**输入**：用户当前的自选股列表（在系统中预配置）

**核心流程**：
1. 系统从行情数据源拉取所有自选股的实时数据
2. 对比当日开盘价 / 昨日收盘价，计算涨跌幅
3. 检测异动状态（涨跌 > 5% / 量比 > 2 等）
4. 渲染为表格，按用户偏好排序

**输出**：
- 自选股列表（最多 30 只）
- 每只股票：代码 / 名称 / 现价 / 涨跌幅 / 涨跌额 / 成交量 / 量比 / 异动标记

**边界条件**：
- 非交易时段 → 展示昨日收盘数据 + 标记"已收盘"
- 数据源故障 → 展示上次缓存数据 + 警告条
- 自选股超过 30 只 → 拒绝添加，提示"请先移除"

**异常处理**：
- 单只股票数据缺失 → 该行展示"数据不可用"，不影响其他股票
- 网络异常 → 自动重试 3 次后展示离线模式

**性能要求**：
- 首屏加载 p95 < 2 秒
- 实时数据刷新延迟 < 30 秒（交易时段）
- 同屏渲染 30 只股票流畅

---

### F2 自然语言 AI 智能选股

**触发**：用户在搜索框输入自然语言查询，按 Enter 或点击"选股"

**输入**：
- 自然语言查询字符串（5-200 字符）
- 示例："最近 3 天放量上涨的新能源股"
- 示例："市盈率小于 20 的银行股"

**核心流程**：
1. 系统接收用户输入
2. 调用 AI 理解用户意图，转换为结构化查询条件
3. 在全 A 股数据中筛选符合条件的股票
4. 按相关性排序
5. 返回前 50 只结果

**输出**：
- 匹配的股票列表
- 每只股票：代码 / 名称 / 关键指标 / 命中原因
- 查询解析（让用户看到 AI 理解的条件）

**边界条件**：
- 查询无法理解 → 返回澄清问题（"你说的'放量'指什么标准？"）
- 结果为空 → 提示"没找到符合的股票" + 建议放宽条件
- 结果过多（>50） → 截断返回前 50 + 提示"还有 N 只"

**异常处理**：
- AI 服务不可用 → 降级到关键词搜索
- 查询超时（>10 秒）→ 返回部分结果 + "查询超时"

**性能要求**：
- p95 < 3 秒出结果
- 准确率 ≥ 80%（人工标注 100 个 query 验证）

**业务规则**：
- 不返回 ST 股票（用户可关闭）
- 不返回近 3 天上市的新股
- 涨停 / 跌停股标记

---

### F3 每日早盘简报（自动推送）

**触发**：定时任务，工作日 8:00 自动触发

**输入**：
- 用户的自选股列表
- 今日大盘数据（开盘前预测）
- 昨日收盘数据 + 重要新闻

**核心流程**：
1. 拉取昨日收盘后到今晨的所有相关数据
2. AI 生成简报（800-1500 字）
3. 按模板组织：大盘回顾 / 自选股动态 / 今日关注 / 行业热点
4. 推送到用户的飞书

**输出**：飞书消息卡片，包含：
- 一句话总结（大盘情绪）
- 自选股状态汇总（涨跌排行）
- 今日 3 个关注点
- 相关新闻链接

**边界条件**：
- 周末 / 节假日 → 不发送
- 用户没有自选股 → 只发大盘简报
- 数据源故障 → 推送"今日简报生成失败"通知，附带错误码

**异常处理**：
- 飞书机器人推送失败 → 重试 3 次，仍失败则记录日志

**性能要求**：
- 8:00 准时推送（误差 < 1 分钟）
- 简报生成耗时 < 30 秒

**业务规则**：
- 不做投资建议，只做信息汇总
- 每篇都含免责声明

---

### F4 异动预警（实时）

**触发**：交易时段（9:30-15:00）每分钟扫描一次

**输入**：用户自选股 + 异动规则

**核心流程**：
1. 每分钟扫描自选股最新数据
2. 检测是否触发异动规则
3. 如果触发，调用 AI 生成异动解读
4. 推送到飞书

**输出**：飞书消息，包含：
- 股票代码 / 名称
- 异动类型 + 数值（涨跌幅 / 量比）
- AI 解读（1-3 句话）
- 操作建议（"关注一下"或"暂不需要操作"）

**边界条件**：
- 同一只股票 30 分钟内不重复推送
- 单日推送上限 10 条 / 用户（防骚扰）
- 涨停 / 跌停只推一次

**异常处理**：
- 数据流中断 → 立即推送"数据流异常"通知
- AI 解读失败 → 用模板化文案兜底

**性能要求**：
- 从触发到推送 < 30 秒
- 推送可靠性 ≥ 99%

**业务规则**：
- 默认异动阈值：涨跌幅 ±5%（用户可调）
- 工作日 9:30-15:00 才推送
- 强制添加免责声明

---

### F5 飞书机器人集成

**触发**：F3 / F4 / F6 等需要推送的场景

**输入**：消息内容 + 用户飞书 Webhook URL

**核心流程**：
1. 构造飞书 Markdown 卡片
2. 调用飞书 Webhook API 推送
3. 记录推送结果（成功 / 失败 / 时间戳）

**输出**：飞书消息 + 推送日志

**边界条件**：
- Webhook 失效 → 通过备用渠道（邮件）通知用户重新配置
- 推送频次超过飞书限制 → 暂存到队列，延后发送

**异常处理**：
- 网络异常 → 重试 3 次（指数退避）

**性能要求**：
- 推送延迟 < 2 秒

**业务规则**：
- 所有推送消息底部加："本消息由 AI 生成，仅供参考，投资有风险"

---

### F6 周度复盘报告

**触发**：定时任务，每周日 20:00 自动触发

**输入**：本周所有交易日数据 + 用户自选股

**核心流程**：
1. 汇总本周自选股表现
2. AI 生成复盘文章
3. 推送到飞书

**输出**：长篇飞书消息，包含：
- 本周大盘走势
- 自选股周度排行
- 关键事件回顾
- 下周关注点

**边界条件**：周末交易所休市，使用最后一个交易日数据

**性能要求**：周日 20:00 推送（误差 < 5 分钟）

## 7. 非功能需求

### 7.1 性能
- Dashboard 首屏 p95 < 2 秒
- 选股查询 p95 < 3 秒
- 推送从触发到送达 < 30 秒
- 系统能承受 50 只自选股的并发监控

### 7.2 可用性
- 整体可用性 ≥ 99%（每月停机 ≤ 7 小时）
- 交易时段（9:15-15:00）可用性 ≥ 99.9%
- 推送可靠性 ≥ 99%
- 非交易时段允许维护

### 7.3 安全
- 不存储任何敏感金融账户信息
- 飞书 Webhook URL 加密存储
- 数据源 API Key 不进 Git，用环境变量管理
- 所有外部 API 调用走 HTTPS

### 7.4 合规
- 不提供投资建议，只做信息汇总
- 所有推送 / 简报附免责声明
- 数据来源标注清楚
- 不爬取付费数据
- v1.0 仅个人自用，不商业化

### 7.5 数据
- 实时数据延迟 ≤ 30 秒
- 历史数据保留 ≥ 1 年
- 每日数据备份

### 7.6 可观测性
- 关键操作日志记录
- 推送成功率监控
- 数据源故障告警

## 9. 验收标准

### AC-F1-01：Dashboard 首屏性能
**Given** 用户有 30 只自选股
**When** 用户访问 Dashboard 页面
**Then** 首屏在 p95 < 2 秒内完整渲染

### AC-F1-02：自选股上限保护
**Given** 用户已有 30 只自选股
**When** 用户尝试添加新股票
**Then** 系统拒绝添加，并提示"自选股已达上限"

### AC-F2-01：自然语言选股准确性
**Given** 用户输入"最近 3 天放量上涨的新能源股"
**When** 系统处理这个查询
**Then** 3 秒内返回符合条件的股票，且准确率 ≥ 80%（人工标注验证）

### AC-F2-02：选股查询无法理解
**Given** 用户输入"今天有什么好股票"（过于模糊）
**When** 系统处理
**Then** 返回澄清问题，不直接给出结果

### AC-F3-01：早盘简报准时推送
**Given** 工作日早上 8:00 到来
**When** 系统自动触发简报生成
**Then** 8:00 ± 1 分钟内推送到用户飞书

### AC-F4-01：异动预警实时性
**Given** 自选股出现涨跌 > 5%
**When** 系统检测到异动
**Then** 30 秒内推送到用户飞书

### AC-F4-02：异动防骚扰
**Given** 同一只股票 30 分钟前已推送过异动
**When** 该股票再次触发同类异动
**Then** 系统不重复推送

### AC-F5-01：推送可靠性
**Given** 系统连续运行 1 周
**When** 统计推送成功率
**Then** 推送成功率 ≥ 99%

### AC-NFR-01：可用性
**Given** 系统运行 1 个月
**When** 统计可用性
**Then** 整体可用性 ≥ 99%

## 11. 度量指标

### 11.1 北极星指标
**用户每天实际使用工具的有效次数**（看 Dashboard / 收推送 / 查选股）

### 11.2 关键指标
**用户价值（核心）**：
- 用户每天手动看盘时长（目标：从 30 分钟降到 5 分钟）
- 重大异动通知到达率（目标：≥ 95%）

**功能使用**：
- 早盘简报打开率（目标：≥ 80%）
- 选股查询日均次数（目标：≥ 3 次）
- 异动推送响应时间（用户查看的速度）

**质量**：
- 系统可用性
- 选股准确率
- 推送可靠性

### 11.3 反指标（防作弊）
- 推送骚扰率（用户禁用推送的比例，目标 < 5%）
- 误报率（异动推送被用户标记"没用"的比例）
