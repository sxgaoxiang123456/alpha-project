04 adversarial-architecture-selection

5 角色法庭辩论 + 6 条反偏见硬约束

⚖️

3 代言人

为各自 owns 的项目写强势辩护书（position paper），按 7 维度深挖，含强制致命缺陷自述。

🔥

1 红队

魔鬼代言人，反对所有候选，主张全自建。对每份 paper 发 3 条带证据的质疑。

📊

1 集成评估师

中立评估多项目复合的工程可行性。产出组合评分表 + Top 3 推荐。

反偏见约束 反什么偏见 实现方式
致命缺陷自述 代言人偏见（self-advocacy） 每份 paper 强制报 3 个缺陷，带证据
随机打乱顺序 位置偏见（positional） Phase 2 papers 伪随机打乱
回应 ≤ 200 字 长度偏见（length） 硬上限，超限视为违规
Lead 禁言 权威偏见（authority） Phase 2 只协调不评价
Phase 1 静默 锚定偏见（anchoring） 严禁 teammate 通信
立场锁定 从众偏见（conformity） 红队/代言人不可倒戈

❌ 普通多 Agent 讨论

  • 无固定立场，自由讨论
  • 温和协商，求同存异
  • 无系统反偏见机制
  • 产出：一段综合结论

✅ 5 角色法庭辩论

  • 角色立场锁定（辩护/反方/中立）
  • 对抗是目的，质疑必带证据
  • 6 条反偏见硬约束
  • 产出：7 维文档 + 辩论记录 + 复用矩阵

05 prd-writer

14 章结构化 PRD · What/How 严格边界

📝

6 核心信息提取

苏格拉底 Q&A 补齐：项目定义、目标用户、MVP 范围、成功指标、NFR、Open Questions。

📐

14 章模板浇筑

每章有目的 + 模板片段 + 反模式。不是自由发挥，是按模具浇筑。

🤖

AI 友好精度

"Specs are the new code"：边界条件、异常处理、验收标准都明确，AI 不需要猜。

章节 必/推/可 核心目的 反模式
1. 文档信息★版本可追溯无版本号 / 永远是 Draft
2. 背景与目标★Why + Non-Goals写成 KPI 罗列
3. 目标用户★画像 + 反画像"所有用户"
4. 用户故事★INVEST 格式成功能列表
5. 功能范围★In-Scope + Out-of-Scope没有 Out-of-Scope
6. 详细描述★触发/输入/流程/输出/边界只写功能名
7. 非功能需求★性能/安全/合规（数字）"系统应该快"
8. UI/交互☆信息架构 + 关键页面写成设计稿评审
9. 验收标准★Given/When/Then"功能跑通即可"
10. 优先级★MoSCoW / RICE全是 P0
11. 度量指标★北极星 + 反指标只看 DAU
12. 依赖约束☆卡在哪 / 等谁缺就绪时间
13. 风险与 OQ★诚实标记不确定假装一切清楚
14. 里程碑☆时间 + buffer太死没 buffer

What / How 一刀切判断法

工程师能换一种实现仍满足该条 → 留在 PRD

工程师不能换实现 → 搬去 TRD

06 speckit-design-injection

Constitution Injection + Selective Re-run

📦

Step 1: Material Placement

把 Stitch/Figma/Claude Design 产出标准化放置。DESIGN.md 放根目录，页面样本放 design-reference/。

⚙️

Step 2: Constitution Injection

调用 /speckit.constitution 追加"Frontend Design System"章节。只写方向，不写 hex 值。

🎯

Step 3: Impact Identification

扫描所有 feature，标记含前端 UI 的。纯后端 feature 不动，避免无效重跑。

🔄

Step 4: Selective Re-run

逐个 feature 重跑 /plan + /tasks。不动 spec/clarify，后端原文保留。每次停下来确认。

错误做法 为什么错 正确做法
推倒全部重来 浪费 6-8 小时，丢失架构决策 只重跑 plan + tasks
重跑 /specify 业务需求没变 spec.md / clarify.md 不动
所有 feature 重跑 纯后端浪费 token，可能回归 只标记含前端的
hex 值写入 constitution 设计迭代时 constitution 失效 constitution 引用 DESIGN.md
// Constitution 是长期契约，DESIGN.md 是当前快照 constitution.md: // 持久，跨迭代 "视觉规范来源是 DESIGN.md" // ✅ 方向 "主色是 #adc6ff" // ❌ 值 —— 设计迭代会改 DESIGN.md: // 快照，可迭代 "Primary: #adc6ff" // ✅ 值写在这里
